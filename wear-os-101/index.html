                        import android.support.v4.app.NotificationCompat;
                        import android.support.v4.app.NotificationManagerCompat;
                        import android.support.v4.app.NotificationCompat.WearableExtender;

                        int notificationId = 001;
                        // The channel ID of the notification.
                        String id = "my_channel_01";
                        // Build intent for notification content
                        Intent viewIntent = new Intent(this, ViewEventActivity.class);
                        viewIntent.putExtra(EXTRA_EVENT_ID, eventId);
                        PendingIntent viewPendingIntent =
                                PendingIntent.getActivity(this, 0, viewIntent, 0);

                        // Notification channel ID is ignored for Android 7.1.1
                        // (API level 25) and lower.
                        NotificationCompat.Builder notificationBuilder =
                            new NotificationCompat.Builder(this, id)
                                .setSmallIcon(R.drawable.ic_event)
                                .setContentTitle(eventTitle)
                                .setContentText(eventLocation)
                                .setContentIntent(viewPendingIntent);

                        // Get an instance of the NotificationManager service
                        NotificationManagerCompat notificationManager =
                                NotificationManagerCompat.from(this);

                        // Issue the notification with notification manager.
                        notificationManager.notify(notificationId, notificationBuilder.build());

Wearable Apps

Just the same as a "normal" Android App but...

  • Use wear emulators or devices
  • Test in different screen sizes
  • Should use the Wear UI Library
  • Distribution: embedded or standalone (or multi-apk)

Wearable Network Access

With Wear OS by Google (Wear 2.0), a watch can communicate with a network directly, without access to an Android or iOS phone.

With Wear 1.x use of Data Layer is needed

Wearable-Handled Communication

  • Wearable Data Layer API
    • Sending and Receiving Messages (MessageClient)
    • Sync Data Items (DataItem)
    • Transfer an Asset (Asset)
    • Transfer large entities or stream data (ChannelClient)
    • Listen:
      • WearableListenerService (for services)
      • OnDataChangedListener (for foreground activities)

Packaging and Distributing

  • multi-apk method is recommended (it works for both 1.x and 2.x)
  • Target watches 
                                
                            <uses-feature android:name="android.hardware.type.watch"/>
  • Standalone? (It can work with an iPhone!!) 
                                
                            <meta-data android:name="com.google.android.wearable.standalone" android:value="true""/>
